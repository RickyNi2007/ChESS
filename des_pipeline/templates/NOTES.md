# LAMMPS Input Template for Reline

This document explains `in.des.template`, the LAMMPS input script used for each
reline state point.

The template describes a 1:2 choline chloride–urea deep eutectic solvent using
OPLS-DES parameters. It tells LAMMPS which files to read, which force equations
to use, how to equilibrate the system, what measurements to collect, and where
to write them.

## How the template becomes a simulation

`pipeline.py` reads `in.des.template` separately for every temperature-density
state point. It replaces:

- `TEMP_PLACEHOLDER` with the target temperature in K;
- `EQ_STEPS_PLACEHOLDER` with the requested equilibration length; and
- `PROD_STEPS_PLACEHOLDER` with the requested production length.

The filled file is saved as `in.des` inside the corresponding
`output/run_.../T..._rho.../` folder. LAMMPS is then started from that folder:

```bash
mpirun -np 4 lmp -in in.des
```

Because LAMMPS runs from the state-point folder, `read_data data.reline` refers
to the copy of `data.reline` in that same folder.

## 1. Units, atom records, and boundaries

```text
units           real
atom_style      full
boundary        p p p
```

### `units real`

This selects the LAMMPS `real` unit system:

- distance: Å;
- time: fs;
- energy: kcal/mol;
- temperature: K;
- mass: g/mol;
- pressure: atm; and
- charge: multiples of the elementary charge.

All force-field parameters and generated data must use compatible units.

### `atom_style full`

This is the atom-record format used for charged molecular systems. Every atom
in the `Atoms` section of `data.reline` has:

```text
atom-ID  molecule-ID  atom-type  charge  x  y  z
```

`full` is needed because reline contains partial atomic charges and
intramolecular bonds, angles, and dihedrals.

### `boundary p p p`

The box is periodic in x, y, and z. An atom that crosses one side of the box
re-enters from the opposite side. Periodic boundaries approximate a small piece
of bulk liquid without physical walls or surfaces.

## 2. Force-field styles

```text
pair_style      lj/cut/coul/long 12.0
bond_style      harmonic
angle_style     harmonic
dihedral_style  opls
special_bonds   lj/coul 0.0 0.0 0.5
```

### `pair_style lj/cut/coul/long 12.0`

This computes nonbonded interactions between atoms:

1. **Lennard-Jones (LJ):** short-range repulsion and dispersion attraction.
2. **Coulomb:** electrostatic attraction or repulsion between partial charges.

The LJ potential is:

$$
E_{\mathrm{LJ}}(r)=4\epsilon
\left[\left(\frac{\sigma}{r}\right)^{12}
-\left(\frac{\sigma}{r}\right)^6\right]
$$

Here, `epsilon` controls the well depth and `sigma` controls the distance scale.
LJ interactions are cut off at 12.0 Å.

`coul/long` splits electrostatics into short- and long-range parts. The pair
style handles the short-range part, while PPPM handles the long-range part.
This is important for an ionic liquid containing choline cations and chloride
anions.

### `bond_style harmonic`

Bond stretching uses:

$$
E_{\mathrm{bond}}=K(r-r_0)^2
$$

`r0` is the equilibrium bond length and `K` is the force constant. Their values
come from the `Bond Coeffs` section of `data.reline`.

### `angle_style harmonic`

Angle bending uses:

$$
E_{\mathrm{angle}}=K(\theta-\theta_0)^2
$$

`theta0` is the equilibrium angle and `K` is its force constant. Their values
come from `Angle Coeffs` in `data.reline`.

### `dihedral_style opls`

Four connected atoms use the OPLS torsional form with coefficients
`V1`, `V2`, `V3`, and `V4`. The coefficients come from `Dihedral Coeffs` in
`data.reline`.

### `special_bonds lj/coul 0.0 0.0 0.5`

Atoms connected through the same molecule already interact through bonded
terms. This command prevents those interactions from being double counted:

- 1–2 atoms (directly bonded): LJ and Coulomb multiplied by 0.0;
- 1–3 atoms (two bonds apart): multiplied by 0.0;
- 1–4 atoms (three bonds apart): multiplied by 0.5.

This scaling is part of the OPLS force-field convention.

## 3. Load the system

```text
read_data       data.reline
```

LAMMPS reads the plain-text data file generated by `scripts/build_data.py`. It
loads:

- the cubic box boundaries;
- atom IDs, molecule IDs, types, charges, and coordinates;
- atomic masses;
- bond, angle, and dihedral connectivity; and
- bonded force-field coefficients.

The nonbonded LJ coefficients are assigned later in this input script.

## 4. Long-range electrostatics

```text
kspace_style    pppm 1.0e-4
```

PPPM means **Particle–Particle Particle–Mesh**. It computes the long-range
Coulomb contribution by mapping charges onto a mesh and solving the
electrostatic problem in reciprocal space.

`1.0e-4` is the requested relative force accuracy. A smaller value requests
greater accuracy but generally requires more computation. PPPM is required
because the selected pair style ends with `coul/long`.

## 5. Atom types and Lennard-Jones coefficients

The integer atom types are assigned by `scripts/build_data.py`:

```text
1 CS    2 HS    3 NA    4 CW    5 HW
6 OY    7 HY    8 CA    9 HA   10 Cl
11 C(urea)  12 O(urea)  13 N(urea)  14 HT  15 HC
```

More detailed chemical meanings are listed in `forcefield/NOTES.md`.

```text
pair_modify     mix geometric
```

Only like-type coefficients (`1 1`, `2 2`, and so on) are listed explicitly.
For unlike types, LAMMPS uses geometric mixing:

$$
\epsilon_{ij}=\sqrt{\epsilon_i\epsilon_j}
$$

$$
\sigma_{ij}=\sqrt{\sigma_i\sigma_j}
$$

Each command has the format:

```text
pair_coeff atom-type atom-type epsilon sigma
```

Under `units real`, `epsilon` is in kcal/mol and `sigma` is in Å. For example:

```text
pair_coeff     10 10  0.1480  3.77
```

sets the chloride–chloride LJ parameters. Electrostatic strength is not set
here; it comes from the atomic charges in `data.reline`.

Types 7, 14, and 15 have zero LJ epsilon and sigma. These hydrogen types have
no LJ interaction but still participate in bonded and electrostatic
interactions according to their parameters.

## 6. Neighbor lists

```text
neighbor        2.0 bin
neigh_modify    delay 0 every 1 check yes
```

LAMMPS avoids testing every possible atom pair at every timestep by keeping a
list of nearby atoms.

- `2.0` adds a 2.0 Å neighbor-list skin beyond the force cutoff.
- `bin` divides space into bins to find nearby atoms efficiently.
- `delay 0` permits rebuilding immediately.
- `every 1` checks on every timestep.
- `check yes` rebuilds only when atom movement makes it necessary.

The skin is a computational setting, not an additional physical interaction.

## 7. Thermodynamic log output

```text
thermo_style    custom step temp pe ke etotal press vol
thermo          100
```

LAMMPS prints the following every 100 timesteps:

- `step`: timestep number;
- `temp`: instantaneous temperature;
- `pe`: potential energy;
- `ke`: kinetic energy;
- `etotal`: potential plus kinetic energy;
- `press`: instantaneous pressure; and
- `vol`: box volume.

This information appears in `log.lammps` and in the terminal output captured as
`experiment.log`.

## 8. Energy minimization

```text
minimize        1.0e-4 1.0e-6 1000 10000
reset_timestep  0
```

Packmol tries to avoid overlap, but its initial coordinates are not an
energy-minimized liquid structure. Minimization moves atoms downhill in
potential energy before molecular dynamics begins.

The four arguments are:

1. energy tolerance: `1.0e-4`;
2. force tolerance: `1.0e-6`;
3. maximum minimizer iterations: `1000`; and
4. maximum force/energy evaluations: `10000`.

Minimization stops when a convergence condition or one of the maximum limits
is reached. `reset_timestep 0` then resets the timestep counter.

## 9. Timestep and initial velocities

```text
timestep        1.0
velocity        all create TEMP_PLACEHOLDER 4928459 mom yes rot yes
```

`timestep 1.0` means that one MD step represents 1 fs.

`velocity all create` gives all atoms initial velocities drawn for the target
temperature. `4928459` is the random-number seed. Keeping it fixed helps make
the velocity initialization reproducible for an otherwise identical starting
configuration.

`mom yes` removes net linear momentum and `rot yes` removes net angular
momentum from the generated velocities.

## 10. NVT equilibration

```text
fix             1 all nvt temp TEMP_PLACEHOLDER TEMP_PLACEHOLDER 100.0
run             EQ_STEPS_PLACEHOLDER
unfix           1
```

`fix nvt` integrates the equations of motion while applying a Nosé–Hoover
thermostat:

- `1` is this fix's identifier;
- `all` applies it to every atom;
- the two identical temperature placeholders keep the target temperature
  constant rather than ramping it; and
- `100.0` is the thermostat damping time in fs.

The number of equilibration steps comes from `eq_steps` in `pipeline.py`.
Equilibration lets the minimized, artificially packed starting structure relax
toward the target temperature before measurements are collected.

`unfix 1` removes this integration fix before production. Only one integration
fix should control the atoms at a time.

The ensemble is NVT:

- **N:** number of atoms is fixed;
- **V:** volume is fixed by the requested density; and
- **T:** temperature is controlled by the thermostat.

## 11. Production setup

```text
reset_timestep  0
fix             2 all nvt temp TEMP_PLACEHOLDER TEMP_PLACEHOLDER 100.0
```

The timestep is reset so production data begins at time zero. A new NVT fix,
with identifier `2`, controls the production trajectory. Measurements are
defined after this line and collected during the production `run`.

## 12. Radial distribution function

```text
compute         rdf all rdf 100
fix             rdf_avg all ave/time 10 100 1000 c_rdf[*] file rdf.out mode vector
```

The RDF, `g(r)`, describes how likely atoms are to occur a distance `r` apart
relative to an uncorrelated system at the same average density.

`compute rdf all rdf 100` creates an aggregate RDF for all atoms using 100
radial bins.

For `fix ave/time`:

- `10`: sample the RDF every 10 timesteps;
- `100`: average 100 samples;
- `1000`: write one averaged block every 1000 timesteps;
- `c_rdf[*]`: write every column produced by the RDF compute;
- `file rdf.out`: output filename; and
- `mode vector`: handle the RDF as vector/array data.

`pipeline.py` reads the last RDF block and uses its radius and `g(r)` columns
to calculate the pairwise excess-entropy proxy.

## 13. Mean-squared displacement

```text
compute         msd all msd com yes
variable        msd_val equal c_msd[4]
variable        time_fs equal step*dt
fix             msd_out all print 100 "${time_fs} ${msd_val}" file msd.dat screen no title "# time_fs msd"
```

MSD measures the average squared distance atoms move relative to their
reference positions.

- `compute msd all msd com yes` computes MSD for all atoms and accounts for
  center-of-mass drift.
- `c_msd[4]` is total three-dimensional MSD.
- `step*dt` converts the current production step into time in fs.
- `fix print 100` writes time and total MSD every 100 timesteps.
- `screen no` prevents these lines from cluttering terminal output.
- `title` writes the header `# time_fs msd`.

The resulting `msd.dat` contains two columns:

```text
time_fs  msd_in_Angstrom_squared
```

`pipeline.py` fits the later part of this data and uses the Einstein relation
to estimate the diffusion coefficient.

## 14. Run production and finish

```text
run             PROD_STEPS_PLACEHOLDER
unfix           2
```

LAMMPS runs the production trajectory for `prod_steps` from `pipeline.py`.
During this run it writes thermodynamic output, RDF blocks, and MSD samples.
`unfix 2` removes the production NVT fix after the run.

## Files used and created for each state point

### Inputs

- `in.des`: the filled copy of this template.
- `data.reline`: box, coordinates, charges, topology, masses, and bonded
  coefficients.

### Outputs

- `experiment.log`: LAMMPS terminal output captured by `pipeline.py`.
- `log.lammps`: standard LAMMPS log and thermodynamic output.
- `rdf.out`: averaged RDF blocks.
- `msd.dat`: production time and total MSD.

## Complete order of operations

1. Select units, atom format, and periodic boundaries.
2. Select nonbonded and bonded force equations.
3. Read the system from `data.reline`.
4. Configure PPPM and LJ parameters.
5. Configure neighbor lists and log output.
6. Minimize the packed starting coordinates.
7. Assign initial velocities.
8. Run NVT equilibration.
9. Reset production time and start the production NVT fix.
10. Define RDF and MSD measurements.
11. Run production while writing measurements.
12. Return control to `pipeline.py` for analysis.
